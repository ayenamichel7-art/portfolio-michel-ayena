# Politique de Sécurité (Security Policy)

Ce document décrit la politique de sécurité pour le portefeuille professionnel de **Michel AYENA**.
En tant qu'ingénieur DevSecOps, la sécurité est une priorité absolue. Ce projet est régulièrement soumis à des audits automatisés (SAST/SCA) et conçu selon les principes de moindre privilège et de sécurisation par défaut (Hardening).

## Versions Supportées

Actuellement, seule la branche principale (`main`) et la dernière version de production sont activement supportées pour les correctifs de sécurité.

| Version | Supportée |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Signalement de Vulnérabilité

Si vous découvrez une vulnérabilité critique ou un défaut de configuration de sécurité, merci de la signaler directement en privé pour permettre une résolution sécurisée (processus de divulgation coordonnée - Coordinated Vulnerability Disclosure).

**Ne pas ouvrir d'issue publique** pour un problème concernant la sécurité informatique de ce projet ou des serveurs de production.

1. Envoyez un email décrivant le problème détaillé à : **Ayenamichel7@gmail.com**
2. Ou contactez-moi directement via **WhatsApp** au +229 01 52 94 41 48

### Ce qu'il faut inclure dans le rapport :
- Une description claire et détaillée de la vulnérabilité
- Les étapes pour la reproduire (PoC si possible)
- Le domaine impacté ou le chemin du fichier spécifique
- L'impact potentiel

Je m'engage à accuser réception de tout signalement dans les 24 heures et à déployer un correctif de sécurité dans les plus brefs délais suivant l'évaluation des risques.
